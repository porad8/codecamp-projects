Introduction

Welcome to a mini documentation of the Julia Programming Language.

In scientific computing, performance and flexibility essential aspects that are achieved with trade-offs which negatively affect performance.

Julia is a flexible and dynamic programming language that mainly eliminates these trade-offs. It takes advantage of modern language design and compiler techniques to provide a single environment productive enough for prototyping and efficient enough for deploying performance-intensive applications with performance comparable to that of traditional statically typed languages.

Once you understand how Julia works, it is easy to write code that is nearly as fast as C/C++. If you find that something is slow, feel free to consult the Perfomance Tips.

Comparison to other languages

Julia features optional typing, multiple dispatch, and good performance, achieved using type inference and jist-in-time (JIT) compilation (and optional ahead-of-time compilation), implimented using low level virtual machine (LLVM). It is multi-paradigm, combining features of imperative, functional, and object-oriented programming,providing ease and expressiveness for high-level numerical computing in the same way as R, MATLAB,and Python, while also supporting general programming.

The most significant departure from typical dynamic languages are:

  • The core language imposes very little: Julia Base and the standard library are written in Julia itself, including primitive operations like integer arithmetic.
  • A rich language of types for constructing and describing objects, that can also be used to make type declarations.
  • The ability to define function behavior across many combinations of argument types via multiple dispatch.
  • Automatic generation of efficient, specialized codefor different argument types.
  • Good perfomance, approaching that of statically compiled languages like C.

In Julia, types are are run-time objects and can also be used to convey information to the compiler. That is, Julia combines both aspects of dynamic languages and statically typed languages adding flexibility.

Advantages of Julia

Julia aims to create an unprecedented combination of ease-of-use, power, and efficiency in a single language. In addition to the above, some advantages of Julia over comparable systems include:

  • Free and open source (MIT licensed)
  • User-defined types are as fast and compact as built-ins
  • No need to vectorize code for perfomance; devectorized code is fast
  • Designed for parallelism and distributed computation
  • Lightweight "green" threading (coroutines)
  • Unobtrusive yet powerful type system
  • Efficient support for Unicode, including but not limited to UTF-8

And many more...

Installation and getting started

Julia installation is straightforward.

Download and install Julia by following these instructions

The easiest way to learn and experiment with Julia is by starting an interactive session by running julia from the command line (remember to skip the dollar sign):

$ julia

To run code in a file non-interactively, you can give it as the first argument to the julia command. For a source file file.jl:

$ julia file.jl

To evaluate expressions written in a source file file.jl, you have to include them into the shell's scope as follows:

include("file.jl")

You can use CTRL-D to exit from the shell or type exit() and press enter.

You can also use the interactive shell as a leraning resource by switching into help mode by pressing ? at an empty julia> prompt before typing anything else. Typing a keyword and pressing enter will fetch the documentation for it.

Variables

A variable, in Julia, is a name associated or bound to a value. It's useful when you want to store a value for later use. For example:

# Assign the value 10 to the variable x
julia> x = 10
10

# Doing math with x's value
julia> x + 1
11

# Reassign x's value
julia> x = 1 + 1
2

# You can assign values of other types, like strings of text
julia> x = "Hello World!"
"Hello World!"

Variable names are case-sensitive and have no semantic meaning (that is, Julia will not treat variable differently based on their names).

Unicode names (in UTF-8 encoding) are allowed.

julia> δ = 0.00001
1.0e-5

julia> 안녕하세요 = "Hello"
"Hello"

In Julia prompt, Unicode symbols (math and others) can be typed by typing the backlashed LaTeX symbol name followed by tab. For example, the variable name δ can be typed by typing:

julia> \delta-tab

You can also copy paste the symbol in help mode to learn more about it.

Variable names must begin with a letter, underscore, or a subset of Unicode points greater than 00A0. Variable names that contain only underscores are immediately discarded after assignment and can't be used later.

While Julia imposes few restrictions on valid names, it is useful to adopt the following conventions:

  • Names of variables are in lowercase.
  • Word separation can be indicated by underscores if the word is hard to read.
  • Names of Types and Modules begin with a capital letter and word separation is shown with upper camel case.
  • Names of functions and macros are in lower case without underscores.
  • Functions that modify their arguments have names that end in !.

For more information about styling see the Style Guide.

Numbers

Numbers are the basic building blocks of arithmetic and computation. Built in representation of such values are called numeric primitives which include integers and floating-point numbers and representations in code are called numeric literals

Julia provides support for Arbitrary Precision Arithmetic, which can handle operations on numeric values that cannot be represented effectively in native hadware representations, though with relatively slower perfomance.

The following are Julia's primitive numeric types:

# 64-bit system:

Strings

Strings are finite sequencies of characters. In Julia, the built-in concrete type used for strings is String which supports the full range of Unicode characters via the UTF-8 encoding.

String literals are delimited by double quotes or tripple quotes, example:

julia> str = "Hello,world."
"Hello, world."

# Breaking long strings:
julia> "This is a long \
line"
"This is a long line"

# String concatenation:
julia> greet = "Hello"
"Hello"
julia> whom = "world"
"world"
julia> string(greet, ", ", whom, ".")
"Hello, world."
julia> greet * ", " * whom * "."
"Hello, world."

# Interpolation:
julia> "$greet, $whom."
"Hello, world."

For an in-depth coverage of strings, see the Strings documentation.

Functions

In Julia, a function is an object that maps a tuple of argument values to a return value. The basic syntax defination of a function in Julia is:

julia> function f(x, y)
x + y
end
f (generic function with 1 method)

# Using a more terse syntax:
julia> f(x, y) = x + y
f (generic function with 1 method)

# calling a function:
julia> f(2, 3)
5

Functions are first class objects in Julia. For a greater coverage of functions, argument passing behavior, methods, multiple dispatch and many more, see the Functions page in the Julia documentation.

Compound expressions

These have a single expression which evaluates several sub-expressions in order, returning the value of the last subexpression as its value.

This is acomplished by using begin blocks and ; chains. Here are some examples:

# begin blocks:
julia> z = begin
x = 1
y = 2
x + y
end
3

# ; chains:
julia> z = (x = 1; y = 2; x + y)
3
Conditional evaluation

Conditional evaluation allows portions of code to beevaluated or not evaluated depending on the value of a boolean expression. Hereis the anatomy of the if-elseif-else conditional syntax:

julia> if x < y
println("x is less than y")
elseif x > y
println("x is greater than y")
else
println("x is equal to y")
end

The ternary operator is closely related to the if-elseif-else syntax used when a conditional choice between single expression values is required. It takes thre operands and it's syntax is a ? b : c where: a is a condition, b is evaluated if a is true and c is evaluated otherwise. The spaces between the operands are mandatory. Here is an example:

julia> x= 1; y = 2; # no output
julia> println(x < y ? "less than" : "not less than")
less than
Loops

Loops are used for repeated evaluation of expressions. In Julia, there are two constructs used; while and for loops. Here are some examples:

# while loop:
julia> i = 1;
julia> while i <= 3
println(i)
global i += 1
end
1
2
3

# for loop:
julia> for j = 1:3
println(j)
end
1
2
3
julia> for i in [1, 3, 7]
println(i)
end
1
3
7

For a exhaustive coverage of the above concepts check the Control Flow page in the Julia documentation.
